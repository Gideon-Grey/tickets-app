# Frontend — учёт внутренних заявок (React + TypeScript)

## Стек
- React 18 + TypeScript
- Vite
- SCSS-модули (CSS Modules + Sass)

## Запуск

```bash
cd frontend
npm install
npm run dev
```

Приложение откроется на http://localhost:5173 и будет обращаться к backend
по адресу из `VITE_API_URL` (по умолчанию `http://localhost:8000`).

Важно: backend должен быть запущен (см. `backend/README.md`), иначе список
заявок покажет состояние ошибки API.

## Сборка

```bash
npm run build
npm run preview
```

## Стили: архитектура

Глобального "монолитного" CSS-файла больше нет — стили разнесены по
SCSS-модулям, привязанным к конкретным компонентам:

```
src/
  index.scss                  только дизайн-токены (CSS custom properties),
                               reset и базовая раскладка #root. Стилей
                               конкретных компонентов здесь нет.
  styles/
    _mixins.scss               миксины брейкпоинтов (tablet/mobile/small),
                               подключаются через @use в модулях компонентов
    shared.module.scss         переиспользуемые атомы: кнопки, поля формы,
                               каркас модального окна — импортируется
                               несколькими компонентами сразу
  components/
    Toolbar.tsx + .module.scss
    TicketTable.tsx + .module.scss
    Pagination.tsx + .module.scss
    Badges.tsx + .module.scss
    CreateTicketModal.tsx       использует только shared.module.scss
    AdminLoginModal.tsx         использует только shared.module.scss
```

Принцип: если стиль используется в одном месте — он живёт в
`Component.module.scss` рядом с компонентом. Если в нескольких — выносится
в `styles/shared.module.scss`. Все размеры, отступы и точки переходов
завязаны на переменные из `index.scss` (`var(--primary)`, `var(--border)`
и т.д.), поэтому смена палитры делается в одном месте.

### Адаптивность / мобильная версия

- `index.scss` задаёт отступы `#root` для трёх диапазонов экрана.
- `TicketTable` рендерит два варианта разметки одновременно: таблицу для
  десктопа (`.tableDesktop`) и список карточек для мобильных (`.cardsMobile`).
  Переключение между ними — чисто на CSS (`display: none` / `display: block`
  по `@include mobile` из `_mixins.scss`), без JS и без лишних перерендеров.
- Toolbar, модальные окна и пагинация перестраиваются в колонку на узких
  экранах; модальные окна на самых маленьких экранах (`@include small`)
  превращаются в нижний bottom sheet вместо центрированного попапа.

## Что реализовано на одном экране

- Таблица заявок: заголовок, описание (с обрезкой), статус, приоритет,
  даты создания/обновления.
- Строка поиска (по `title` и `description`, debounce 300мс).
- Фильтры по статусу и приоритету.
- Управление сортировкой (по дате создания / по приоритету, возрастание/убывание).
- Форма создания заявки (модальное окно) с клиентской валидацией
  (заголовок 3–120 символов, описание до 1000 символов) и показом ошибок API.
- Изменение статуса прямо из строки таблицы (выпадающий список); для заявок
  в статусе "Выполнена" список заменяется на статичный бейдж — менять статус
  и удалять такие заявки нельзя.
- Удаление заявки — кнопка видна только после входа администратора.
- Вход администратора (модальное окно, дефолтные креды `admin` / `admin`).
  Токен хранится в `localStorage` и переживает обновление страницы.
- Состояния загрузки, пустого списка и ошибок API отображаются в области таблицы.
- Пагинация с выбором размера страницы.

## Структура проекта

```
frontend/
  src/
    api/client.ts            обёртка над fetch, разбор ошибок API
    components/
      Toolbar.tsx + .module.scss        поиск, фильтры, сортировка, шапка
      TicketTable.tsx + .module.scss    таблица/карточки + loading/empty/error
      Pagination.tsx + .module.scss     пагинация
      CreateTicketModal.tsx             форма создания заявки
      AdminLoginModal.tsx               вход админа
      Badges.tsx + .module.scss         статус/приоритет, общие справочники
    styles/
      _mixins.scss             миксины брейкпоинтов
      shared.module.scss       общие атомы: кнопки, поля форм, модалка
    App.tsx                   состояние, оркестрация запросов
    types.ts                  типы, совпадающие со схемами backend
    index.scss                дизайн-токены, reset, базовая раскладка
    main.tsx
```
